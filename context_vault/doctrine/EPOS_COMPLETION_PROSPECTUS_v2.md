## 1.2 The 10-Layer Action Matrix (Applied at Every Touchpoint)

1. **Identity** — Resolve who this is (PS-EM, R1, VAULT → Lead Intake & Scoring)
2. **Intent** — Classify psychological state: Discovery/Education/Decision (TTLG, MA1 → Pre-Classifier)
3. **Data Capture** — Normalize into Vault fields and tags (VAULT, META, BUS → Vault Adapter)
4. **Classification** — Map to niche, revenue layer, node bundle (TTLG, S1, Research → Node Mapping Engine)
5. **Scoring** — Assign Resonance Score and Steward Threshold (AN1, MA1, BI → Echolocation Engine)
6. **Routing** — Choose next touchpoint and sequence (META, BUS, PS-EM → Journey Router)
7. **Delivery** — Compose and send governed asset (A1, V1, PS-EM, M1 → Email/Content Asset Engine)
8. **Feedback** — Log opens, clicks, reply semantics (PS-EM, AN1, VAULT → Feedback Collector)
9. **Learning** — Update weights and templates (AN1, MA1, Feature Proposer → BI/Tweak Node)
10. **Steward Signal** — Decide whether to ping Growth Steward (Growth Steward, TTLG, BI → Steward Gate)

## 2.2 Voice Agent Roles

- **Lead Qualifier**: Warm, professional. Inbound calls → classify tier → book diagnostic. Escalate: score > 85 or enterprise.
- **Diagnostic Guide**: Authoritative, patient. TTLG walkthrough. Escalate: complex objection or custom pricing.
- **Onboarding Assistant**: Friendly, structured. Setup + requirements. Escalate: scope change.
- **Support Agent**: Empathetic, solution-focused. Troubleshoot + route. Escalate: SLA breach.

## 2.3 Constitutional Voice Guardrails

- Every script passes Brand Validator before deployment
- No financial promises, guaranteed outcomes, or competitor disparagement
- All calls transcribed locally (zero cloud upload)
- GRAG suggestions are advisory only
- Human escalation mandatory: 3x failed objections, > $5K/mo deal, or prospect requests human

## 2.4 The GRAG Loop

LISTEN → ANALYZE → SUGGEST → LEARN → EVOLVE

Between-session intelligence auto-generates: follow-up actions, briefing materials, journey position updates.

# PART VI: I.I.D.E.A.T.E. v2 — THE COMPULSION FRAMEWORK

> Implementation without documentation is invisible. Documentation without surveillance is fiction. Surveillance without recursion is stagnation.

## The Triple Compulsion (Every Stage)

1. **IMPLEMENT** — Produce a concrete artifact. No planning without building.
2. **DOCUMENT** — Log with rationale, context, and metrics. Undocumented = invisible.
3. **SURVEIL** — Measure against targets. Unsurveilled = drifting.

## Stage Requirements

### IMMERSION
- IMPLEMENT: immersion_brief.json
- DOCUMENT: Vault storage with mission_id linkage
- SURVEIL: Completeness score ≥ 90%

### IDEATION
- IMPLEMENT: 3+ distinct candidates
- DOCUMENT: ideation_registry.json with all candidates + elimination reasoning
- SURVEIL: Diversity score ≥ 3 candidates

### DESIGN
- IMPLEMENT: Complete specification (zero empty fields)
- DOCUMENT: Spec + 3+ pre-mortem failure scenarios
- SURVEIL: Completeness = 100%. Incomplete = Gate violation.

### EXECUTION
- IMPLEMENT: Deployable artifact + validation pass + Event Bus emission
- DOCUMENT: Execution receipt with artifact_id, platform, timestamp
- SURVEIL: Completion rate ≥ 85% first attempt

### ANALYSIS
- IMPLEMENT: Day 1 Reflection + 48-Hour Dashboard
- DOCUMENT: context_vault/bi_history/ with predicted vs actual
- SURVEIL: Coverage ≥ 95% of artifacts with 48hr data

### TWEAKING
- IMPLEMENT: Weight adjustment + impact prediction
- DOCUMENT: tweak_log.jsonl with parameter, old/new values, trigger, prediction
- SURVEIL: Effectiveness ≥ 60% producing predicted improvement

### EVOLUTION
- IMPLEMENT: 1+ template AND 1+ anti-pattern. Score > 85 → product eval within 7 days.
- DOCUMENT: evolution_manifest.json with templates, anti-patterns, proposals
- SURVEIL: Output ≥ 2 new artifacts per cycle. Vault growth measured.
