## V. Operating Cadence — Both Heartbeats

The canonical flows are defined in `../councils/OPERATING_PROTOCOL_v1.md`:

- **Marketplace Launch (Primary):** Oracle researches → Architect plans → Forge builds → Chronicler synthesizes → Steward delivers → Sentinel audits.
- **Build Track (Support):** Sentinel identifies gaps → Architect writes directives → Forge executes → Oracle validates → Chronicler documents → Steward monitors.
- **Decision Protocol:** Jamie states intent → Architect structures options → Oracle provides market evidence → Sentinel flags blind spots → Jamie decides → Forge executes → Steward tracks → Chronicler records for training.
- **Conflict Resolution:** Architect presents both positions with evidence → Jamie makes the sovereign decision → logged to PM surface → Reward Bus scores outcome at 90 days.

**Heartbeat synchronization:**
- Marketplace evidence (signed deals, reviews, conversations) feeds the build via Steward BI hooks → Chronicler synthesis → Sentinel audit → Architect Directives → Forge improvements.
- Build outputs (new nodes, refactored agents) return to marketplace surface via Forge receipts → Steward client-facing updates → Oracle re-validation → Chronicler re-synthesis → Architect Directive adjustment.
