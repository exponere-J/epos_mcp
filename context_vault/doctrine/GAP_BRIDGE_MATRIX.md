## Preamble

The Gap Bridge Matrix is the **Rosetta Stone** between what EPOS can do
(proprioception, from `organism_state.json` `capability_index`) and what
the market demands (Oracle research via Firecrawl + competitive intel).

The matrix has six columns:

1. **Capability cluster** — the tag from `capability_index`
2. **Components** — count of operational components carrying that tag
3. **Lead component** — the canonical entrypoint (component_id)
4. **Market gap addressed** — short prose: what buyer pain this resolves
5. **Market alignment** — `STRONG | MODERATE | WEAK | TBD` (filled by Oracle)
6. **Action** — `SHIP | EVALUATE | BUILD | DEFER`

Once Oracle's Firecrawl scan completes, column 5 populates. Until then,
the 10 Sovereign-pre-confirmed bridges below are seeded; everything else
is `TBD`.

# APPENDIX A — Research-to-Merge Protocol

The Gap Bridge Matrix surfaces opportunity. The Research-to-Merge Protocol
operationalizes adoption. Six steps:

```
1. DISCOVERY (Oracle + Firecrawl)
   Market research surfaces a framework, tool, or approach
   that addresses a gap our products don't cover.

   Example: "Buyers want interactive prompt builders, not just templates.
   Competitor X has a React-based prompt builder on their site."

2. EVALUATION (Sentinel + TTLG)
   Sentinel asks: "Can we build this with what we have?"
   TTLG Thinker scores: compatibility (does it fit our architecture?),
   effort (how many agentic hours?), revenue (does it increase product value?),
   sovereignty (does it create a new dependency?)

   Scoring: each dimension 1-10. Composite > 7 = MERGE.
   Composite 4-7 = EVALUATE FURTHER. Composite < 4 = SKIP.

3. SIMULATION (MiroFish, when active)
   Before merging, simulate: 100 virtual buyers compare
   our product WITH the merged feature vs WITHOUT.
   Does purchase intent increase? By how much?

4. INTEGRATION (Forge)
   Desktop Code builds the feature into the product.
   Governance watermark. Pre-mortem check. Doctor validation.

5. REPACKAGING (Chronicler)
   NotebookLM generates updated Audio Overview for the product.
   Listing copy adjusted. Price point re-evaluated if value increased.

6. MEASUREMENT (Steward + Echolocation)
   After launch: does the merged feature improve conversion?
   Echolocation scores at T+24h and T+48h.
   Reward Bus logs the signal. QLoRA trains on the delta.
```

## Threshold rules

- **Composite ≥ 7** → MERGE (proceed to Step 3+)
- **Composite 4–6** → EVALUATE FURTHER (Sentinel deep-dive; Sovereign decision)
- **Composite < 4** → SKIP (log to `context_vault/bi/research_to_merge_skipped.jsonl`)
