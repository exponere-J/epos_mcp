The GRAG Architecture — What Must Be Specified
GRAG = Graph-Retrieval Augmented Generation — Constitutional Definition

GRAG tools attached to Phi-3 Mini will query the Context Vault, fire events on the Event Bus, and write results to node outputs. This means Phi-3 is no longer just a local LLM — it becomes an orchestration participant with read/write access to the constitutional ecosystem. The Constitution v3.1 amendment must define: (1) which Context Vault paths are readable by GRAG tools, (2) which Event Bus topics GRAG tools can publish to, (3) whether GRAG outputs require Governance Gate validation before writing to Vault, and (4) tool failure behavior — if internet search fails, does the GRAG chain degrade gracefully or hard-fail the mission?

Google Sheets Implementation — Identical Schema Rules
Pre-Mortem: What Breaks Schema Identity
  • Auto-formatting: Google Sheets will auto-format dates as MM/DD/YYYY — AirTable stores ISO 8601. Fix: force all date columns to Plain Text format until export.
  • Currency symbols: Sheets adds $ symbol which breaks numeric comparisons. Fix: store numbers only in both systems, apply formatting display-only.
  • Column ordering: Any manual column reorder in Sheets breaks the sync script. Fix: lock sheet structure via sheet.protect() — read-only for schema columns.
  • Empty strings vs null: AirTable nulls become empty strings in Sheets. Fix: n8n sync script normalizes null → empty string on ingest.
  • Record IDs: AirTable Record IDs (recXXXXXX) must be preserved in Sheets as a reference column. Never delete this column.

The Separation Rule — Written in Constitution
Constitutional Amendment: Personal / Business Data Firewall
  • Article XIII (proposed): No EPOS operational node shall read from, write to, or reference any field in the LIFE_Command_Base or personal vault namespace. Violation = constitutional breach, immediate quarantine.
  • Personal AirTable base uses separate API key — EPOS agents do not have access to this key. It exists only in Jamie's personal .env, not in the EPOS workspace .env.
  • Trust and estate data (TRUST_ tables) live in LIFE_Command_Base, not EPOS_Command_Base. The two bases may share a Sheets mirror owner (your Google account) but have zero programmatic cross-access.
  • PGP Property Solutions uses PGP_Command_Base — completely separate from EPOS. The shared-business nature with Stacey requires its own data governance (Stacey has read access to PGP tables, not EPOS tables).
