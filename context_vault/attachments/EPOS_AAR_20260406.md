## 4. What Went Wrong

- **AAR was not written in the initial session.** The session close protocol ran (Doctor, heal, diagnostic, dashboard) but produced a summary table, not an AAR. This is the governance failure that prompted the AAR Constitutional Amendment. The summary says what was built; the AAR captures what was learned.
- **BrowserUse assumption failed.** The directive assumed BrowserUse would be available for automated LinkedIn/X posting. It was not installed. The M1 Publisher handled this with a clean fallback, but the directive should have included a pre-mortem scenario for this.
- **FOTW scanner-to-CCP integration uses Groq LLM rather than the local CCP Engine.** The element_router.py calls Groq for element extraction rather than importing from workspace/ccp/ccp_engine.py. This means the parser is cloud-dependent rather than sovereign. Future session should wire CCP Engine directly.
- **Doctor checks for TTLG props validity and self-healing responsiveness were documented but not implemented.** The certification mission registered the nodes but did not add the specific health checks to epos_doctor.py.

## 5. What Was Learned

- **The AAR must be a constitutional mandate, not a suggestion.** Session close protocols are treated as verification (did it pass?), not reflection (what did it teach?). The fix is the AAR Constitutional Amendment now ratified and stored.
- **Directive Scenario Projection Blocks prevent environment assumptions.** BrowserUse availability should have been a pre-mortem scenario. Every future directive includes 3-5 failure scenarios with expected behavior.
- **LangGraph conditional edges map naturally to business logic.** The GO/PIVOT/KILL gate pattern is exactly what LangGraph was designed for. The TTLG pipeline is a canonical LangGraph application.
- **Mirror Report TTS summaries must be tested against actual Audio Overview generation.** 154 words is within limits, but we have not yet tested a NotebookLM import of the summary file.
- **The 9th Order gap tracker transforms imagination into a prioritized build queue.** By recording Layer 13 gaps with feasibility and market impact scores, the system converts "what if" into "build next."
