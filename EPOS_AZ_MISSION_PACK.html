AZ-GS-CLIENT-QUERY

Growth Steward Client-Facing Query Interface

Tier: GS · Depends on: AZ-AT-BRIDGE-001, AZ-CC-ADVISORY-TAB

Design

Clients can ask EPOS questions directly, but only in governed domains. Every query creates an Advisory Request row (GS tier) and responses are grounded in Airtable data only.

Allowed Questions

"What should I post about next week?" · "Where am I wasting the most time?" · "What did last week's analytics actually say?" · "How is my book club growing?"

Blocked Questions

"Should I pivot my whole business?" · "Should I hire/fire X?" · Any question outside content, friction, analytics, or basic strategy scope.

Execution Plan

Create intake channel: For HITL phase, a simple Google Form or Discord command that writes to Advisory Requests with Inquiry Tier = "GS"
Build GS response generator: A prompt template that takes the question + reads from Content Calendar, Shadow Log, Soil Samples, and DM Stewardship via Airtable API. Returns 2-3 concrete options + one "do nothing" baseline, each tied to COML/OC or Missed Revenue numbers.
Store prompt template in context_vault/prompts/gs_query_response.txt
Response boundary enforcement: If the question doesn't match allowed categories (content, friction, analytics, strategy), return: "Great question — that's one I want to think through with you personally. I'll flag it for our next touchpoint." And set Inquiry Tier to "SA" for human review.
Log every interaction as an Advisory Request row for BI analysis.

AZ-SA-HITL-FLOW

Steward Advisor Option Generation + Approval Flow

Tier: SA · Depends on: AZ-CC-ADVISORY-TAB

Design (Flow)

1. Client question or GS escalation arrives → Advisory Request row created (Status: Pending)
2. EPOS reads Context Snapshots (Shadow Log + Soil Samples for that creator)
3. EPOS generates 3 options (Conservative / Aggressive / Experiment) with Impact Delta
4. Status → "Options Generated" → appears in Command Center Advisory Tab
5. Jamie reviews, picks/edits → sets Chosen Option + Approved By/At
6. Status → "Approved" → event emitted → downstream actions fire
7. Resolution logged after execution → Status → "Closed"

Execution Plan

Build option generator prompt: Takes question + creator's latest Shadow Log (top 3 by Priority) + latest Soil Sample. Outputs JSON with 3 options + Impact Delta.
Store in context_vault/prompts/sa_option_generator.txt
Wire to Advisory Requests: On new row with Status="Pending", run option generator, write results to Option A/B/C + Impact Delta fields, set Status="Options Generated"
Approval → Event: When Jamie clicks Approve in Command Center, emit advisory.option.approved event. Downstream subscribers (defined in AZ-EVENT-WIRING) handle execution.

AZ-VOICE-PERSONA

Steward Voice Constitutional Document + Channel Adapters

Tier: GS · Depends on: AZ-AT-BRIDGE-001

Design

One personable communication layer that shows up everywhere. The Steward Voice is the constitutional personality — consistent across channels, adaptable per creator's voice profile.

Voice Constitution (stored in Context Vault)

Tone: Warm, grounded, agricultural. Peer-level collaborator, not boss. Uses "we" language. Celebrates consistency, never compares creators to each other.

Cadence: Crop Reports weekly. Posting alerts as-needed. Decision recaps after Advisory approvals. Never more than 2 unsolicited touches per day.

Boundaries: Never generates opinions the creator didn't express. Never pitches EPOS unless directly asked. Never exposes raw data without interpretation. Never uses comparison anxiety.
